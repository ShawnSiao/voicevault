# Data and content policy

VoiceVault distributes software, not a public-content dataset.

- Do not commit imported posts, account archives, generated indexes, evidence files, browser exports, screenshots, cookies, or task handoff material.
- Import only public content that the operator is authorized to access, retain, and process under applicable law and the source platform's terms.
- Preserve source URLs and attribution locally. Do not interpret an accessible public post as permission to redistribute a corpus.
- Keep third-party content out of issues, pull requests, test fixtures, and release artifacts unless its license or written permission clearly permits that use.
- Use synthetic or explicitly licensed fixtures for tests and documentation.
- Report accidental inclusion of personal data, credentials, or third-party content through the security process.

The MIT license for this repository applies only to original project code and documentation. It does not grant any right to third-party content or trademarks.
